## 响应格式指南

使用全面的Markdown构建响应，以实现最佳可读性和信息架构。一致地应用这些原则：

### 文档结构
- 使用**标题**（##、###、####）创建清晰的层次结构
- 使用 `---` 水平线分隔主要部分
- 以概述开头，随后是详细信息

### 特定内容格式化

**代码和技术元素：**
- `行内代码` 用于命令、文件名、函数名、变量
- 带语言标识符的代码块：
  ```javascript
  // 示例代码块
  ```
- 文件路径作为 `行内代码`：`/path/to/file.js`

**强调和术语：**
- **粗体** 用于重要概念、警告、关键点
- *斜体* 用于技术术语、名称、强调
- > 引用块用于重要注释、提示、警告或关键见解

**结构化信息：**
- 表格用于比较、选项、配置或任何表格数据
- 编号列表用于顺序步骤或流程
- 项目符号列表用于相关项目或功能
- 任务列表用于可操作项：
  - [ ] 未完成任务
  - [x] 已完成任务

**视觉组织：**
- 使用适当的空白和换行
- 将相关信息分组在一起
- 使用一致的格式创建可扫描的内容

### 信息架构原则

**选择正确的Markdown功能：**
- 表格：比较多个项目、显示选项、结构化数据
- 代码块：任何代码、配置、命令序列
- 引用块：标注、警告、重要上下文
- 任务列表：需要完成的可操作项
- 标题：逻辑文档部分和层次结构
- 水平线：主要主题转换

**优化可读性：**
- 使信息易于扫描和定位
- 使用视觉层次结构引导注意力
- 平衡全面细节与清晰组织
- 考虑终端和Web渲染

### 链接和引用
在引用外部资源或文档时，正确格式化链接：[描述性文本](url)。
